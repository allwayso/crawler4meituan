# 组件规格说明书 (Component Specifications)

本文件定义了自动化采集流程所需的各个组件及其功能规格。

### 0. LLM 配置组件 (`config_manager.py`)
*   **功能：** 统一管理 API 配置（如 API Key、Base URL）、模型参数以及所有组件共享的依赖库配置。
*   **输入：** 无（从配置文件读取）。
*   **输出：** 提供全局配置对象供其他组件调用。

### 1. 餐厅列表获取组件 (`restaurant_detector.py`)
*   **功能：** 截图并识别当前屏幕上的餐厅列表，结合 OCR 与 LLM 进行清洗。
*   **输入：** 无（内部调用截图与 OCR）。
*   **输出：** 返回清洗后的餐厅列表（包含名称、坐标、是否已访问标记等信息）。

### 2. 元素识别组件 (`element_detector.py`)
*   **功能：** 在当前页面识别目标文字（如“菜品”、“查看全部”等）。
*   **输入：** 目标文字字符串。
*   **输出：** 返回目标元素的中心坐标。
*   **异常：** 若未找到目标文字，抛出 `ElementNotFoundException`。

### 3. 菜单识别组件 (`menu_detector.py`)
*   **功能：** 接受两张截图，执行 OCR 识别和 LLM 清洗，返回 JSONL 格式的菜单数据。
*   **输入：** 两张截图路径。
*   **输出：** JSONL 格式的菜单数据。

### 4. 流程控制组件 (`workflow_manager.py`)
*   **功能：** 协调上述所有组件，执行完整的自动化采集循环。
*   **工作流：**
    1. 调用 `restaurant_detector` 获取餐厅列表（包含 `is_visited` 标记）。
    2. 遍历列表：
       - 若有未访问餐厅，保存餐厅信息变量，点击 `(500, y+100)` 进入详情页。
       - 若无未访问餐厅，执行滑动 `900`。
    3. 调用 `element_detector` 查找“菜品”：
       - 若抛出异常，`break`。
       - 若找到，点击返回坐标。
    4. 调用 `element_detector` 查找“查看更多”：
       - 若抛出异常，执行返回。
       - 若找到，点击返回坐标。
    5. 调用 `element_detector` 查找“网友推荐”：
       - 若抛出异常，执行两次返回。
       - 若找到，点击返回坐标。
    6. 调用 `menu_detector` 获取菜单。
    7. 将菜单信息与餐厅信息变量整合，写入 `D:\crawler\data\restaurants.json`。
    8. 执行两次退出，返回步骤 2。
